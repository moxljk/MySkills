# Current State

Use a lightweight tracking view only when it helps the user stay oriented.

Read this file only when ordinary periodic summaries are not enough to keep the current direction visible.

Choose whatever form best fits the conversation and environment.

Keep it small:
- one current main question
- one to three unresolved sub-questions
- optionally one active question marked as the current focus

Good examples:
- "What is the real question here?"
- "Is the problem mainly about logic, evidence, or framing?"
- "Which of the two directions matters more right now?"

Avoid:
- turning the tracking view into a project implementation plan
- adding every minor thought the user mentions
- keeping resolved items around after they are clear

Use the tracking view to support conversation, not replace it.
