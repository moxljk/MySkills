# Reference Index

Use this file as a lightweight index, not a checklist.

Read only the section that matches the current conversation. Do not load every reference by default.

## Reference Map

Read order and purpose:

| File | Purpose | Read When |
| --- | --- | --- |
| `reference-index.md` | Navigation only | Start here when you need reference help |
| `confusion-patterns.md` | Diagnose what kind of confusion is present | You are not yet sure what the user is stuck on |
| `question-types.md` | Improve the wording of the next move | You already know the move and want sharper phrasing |
| `current-state.md` | Keep the current direction visible | Ordinary summaries are not enough to keep the user oriented |
| `example-scenarios.md` | Calibrate by example | You want quick examples of the broader job this skill covers |

## How To Navigate

- If you are not yet sure what kind of confusion you are looking at, read [confusion-patterns.md](./confusion-patterns.md) first.
- If you already know what kind of move you want to make and just want better ways to phrase it, read [question-types.md](./question-types.md).
- If the user would benefit from a visible way to track what is still unresolved, read [current-state.md](./current-state.md).
- If you want a quick sense of the kinds of situations this skill can cover, read [example-scenarios.md](./example-scenarios.md).

## Decision Rule

Use `confusion-patterns.md` to identify the situation.
Use `question-types.md` to sharpen the next question or reframing once the situation is clear enough.
Use `current-state.md` only when keeping the current direction visible would help the user think.
Use `example-scenarios.md` only for orientation, not for decision rules.

## Core Rule

Use references to sharpen the next move, not to force the conversation into a taxonomy.
