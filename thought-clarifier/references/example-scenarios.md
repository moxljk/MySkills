# Example Scenarios

Use this file only for orientation. These are examples of situations the skill can handle, not an exhaustive list.

This file is for calibration by example, not for deciding the next move.

## Unclear Idea Or Need

The user has an initial idea but does not yet know what they really want.

Example:
- "I want to make a skill for clarifying thoughts, but I do not yet know what that skill should really do."

## Large Task, No Clear Starting Point

The user has received a large task and feels overwhelmed. They need help breaking it into meaningful parts, understanding dependencies, and finding a sensible place to begin.

Example:
- "I need to add a new scheduling algorithm into a large inference framework, but I do not know how to break the task down or where to start."

## Vague Sense Of Value Or Flaw

The user feels that something is important, promising, inconsistent, or wrong, but cannot yet explain why.

Example:
- "I think there is a logical issue here, or maybe an important opportunity, but I cannot yet put it into words."

## Competing Directions

The user sees multiple directions and needs help understanding which one should be clarified first.

Example:
- "There are several ways I could approach this, and I cannot tell which direction matters most right now."

## Core Reminder

These scenarios are examples of one broader job:
- identify what most needs clarification
- turn it into a clearer structure
- help the user continue from there
